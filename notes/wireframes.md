## Org AI features ‚Äî UX flow and ASCII wireframes

This document maps the UX for organization‚Äëscoped AI configuration, usage logging, and admin/superadmin management. It aligns with the implementation plan in `notes/plan.md` and follows the repo guardrails (Next.js App Router, Tailwind v4 + shadcn/ui, Node runtime for AI routes, CSRF, JWT, Sonner toasts).

### Roles and access

- Superadmin: Manage any organization‚Äôs AI (providers, models, defaults, logs, purge). Bypasses membership.
- Org admin: Manage own org‚Äôs AI (providers, models, defaults, logs, purge).
- Org member: Can generate via org defaults and view ‚ÄúMy AI Usage‚Äù for their own logs only.

### Primary routes

- Admin
  - /admin/organizations/[orgSlug]/ai
    - Providers: status/verify/upsert/delete
    - Usage: filterable logs, totals, row detail, purge
- Organization settings (admins)
  - /o/[orgSlug]/settings/ai/providers
  - /o/[orgSlug]/settings/ai/usage
- Member usage
  - /o/[orgSlug]/ai/my-usage
- API (Node runtime only)
  - /api/orgs/[orgSlug]/ai/providers/\*
  - /api/orgs/[orgSlug]/ai/models/\*
  - /api/orgs/[orgSlug]/ai/generate
  - /api/orgs/[orgSlug]/ai/logs/\*

## Flow overview

1. Provider key management

- Open Providers ‚Üí Manage provider ‚Üí Verify key ‚Üí Upsert/save ‚Üí Optionally add models ‚Üí Set default model

2. Model allowlist and defaults

- Add model (provider/name/label/max tokens) ‚Üí Clamp to provider caps ‚Üí List appears ‚Üí Set default (one per provider per org)

3. Generation

- Member action in app (feature = "generic-text" initially) ‚Üí API /generate resolves org config ‚Üí AI SDK call (SSE optional) ‚Üí Log usage ‚Üí Response/stream to client

4. Usage visibility and hygiene

- Admins: Org Usage page with filters, totals, pagination, purge dialog
- Members: My AI Usage (self‚Äëscoped), no purge

5. Safety rails

- CSRF on mutations; RL 30/min user, 60/min IP; redaction + truncation of logs; never expose API keys client‚Äëside; Node runtime only

## Key flows (sequence diagrams)

### A) Upsert provider key

```
User(Admin) ‚Üí UI: Open Manage (Gemini)
UI ‚Üí API: POST /providers/verify { provider, apiKey }
API ‚Üí UI: 200 { verified: true }
UI: Enable Save
UI ‚Üí API: POST /providers/upsert { provider, apiKey }
API: encrypt key, store, lastVerifiedAt=now, audit
API ‚Üí UI: 200 { ok: true }
UI: toast.success("Gemini key saved")
```

Errors

- 400 invalid key ‚Üí toast.error(‚ÄúInvalid API key‚Äù)
- 429 (verify RL) ‚Üí toast.error with Retry‚ÄëAfter

### B) Add model and set default

```
UI ‚Üí API: POST /models { provider, name, label?, maxOutputTokens }
API: clamp tokens per provider cap; create/update; audit
API ‚Üí UI: 200 { model }
UI: toast.success("Model added")
UI ‚Üí API: POST /models/set-default { provider, modelId }
API: ensure single default per (org, provider); audit
API ‚Üí UI: 200 { ok: true }
UI: toast.success("Default set")
```

Guard: DELETE /models/:id fails if default ‚Üí prompt to reassign default first.

### C) Generate (SSE)

```
Client(Member) ‚Üí API: POST /generate { feature, prompt, stream: true }
API: checkAndRecord RL; resolve config; startLog; call AI; stream SSE
SSE events: token, usage, error, done (echo x-correlation-id)
API: finishLog(tokens, latency, status)
Client: render streamed tokens; handle usage and done; show error toast if any
```

SSE event examples

- event: token data: { text: "‚Ä¶chunk‚Ä¶" }
- event: usage data: { tokensIn: 123, tokensOut: 456 }
- event: error data: { code: "AI_CONFIG_MISSING_API_KEY", message: "‚Ä¶" }
- event: done data: { ok: true }

Rate limit

- 429 Too Many Requests with Retry‚ÄëAfter seconds header; UI shows toast and suggests retry.

## States and edge cases

- Missing provider key ‚Üí generation fails with AI_CONFIG_MISSING_API_KEY; link to Providers page.
- Stale verification (>7 days) ‚Üí soft re‚Äëverify on use; if invalid, return config error.
- No default model for provider ‚Üí generation fails with AI_CONFIG_MISSING_DEFAULT_MODEL.
- Model tokens exceed provider cap ‚Üí clamp in API; show info tooltip in UI.
- Logs redaction ‚Üí emails, URLs, long IDs; truncate to AI_LOG_TRUNCATE_MAX_BYTES; never store API keys.
- CSRF required on state changes; Origin/Referer enforced.
- Edge runtime is never used for AI/DB; only Node runtime.
